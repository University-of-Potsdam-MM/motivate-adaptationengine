Class: AdaptationEngine

AdaptationEngine

new AdaptationEngine(noolsDSL)

AdaptationEngine.js, line 45

The adaptation engine encapsulates the rule engine and the context detection. Via callbacks an application can listen to events fired by the rule engine during the rule evaluation process and react to the triggered rules.

The adaptation engine and it's components require the following external libraries:

  • jQuery
  • nools.js & its constraint parser (lib/parser/constraint/parser.js)
  • require.js
  • ease.js
Name Type Description
noolsDSL String

The adaptation rules as nools DSL format.
Example
$.ajax({
    url: "www.domain.com/pathToRuleGenerator?params",
    success: function(noolsDSL) {
        var adaptationEngine = new AdaptationEngine(noolsDSL);
    }
});

startRuleMatching(intervalInMilliseconds)

AdaptationEngine.js, line 167

Starts the rule matching process. If an integer values is provided the rule matching process will repeat every x milliseconds until it is halted manually by executing AdaptationEngine#stopRuleMatching.

Name Type Description

restrictFeatureCallback(feature, contextInformation)

AdaptationEngine.js, line 63

The callback returns with a string containing the feature to be restricted and an array of context information that triggered the rule.

Name Type Description
feature string

The feature to be restricted.
contextInformation Array.<ContextInformation>

An array of context information that triggered the rule.
Example
adaptationEngine.setRestrictFeatureCallback(function(feature, contextInformation) {
    console.log("Restrict Feature -> "+feature); // Restrict Feature -> AppUsageFeature
    for(var index in contextInformation) {
        console.log(contextInformation[index].description()); // see ContextInformation#description
    }
});
